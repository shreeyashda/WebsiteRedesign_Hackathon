Project: Centauri Platform Redesign (Hackathon)

Goal:
Redesign and prototype Centauri’s digital presence as a dual-sided platform for creators and brands. The objective is to improve clarity, user journeys, engagement, and conversion through strong UX, UI, and interaction design.

Context:
Centauri is positioned as a “growth partner” for creators and brands. However, the current experience lacks:

* Clear product definition
* Defined user journeys
* Strong CTA hierarchy
* Conversion-focused structure

Centauri wants to also work on their solutions service, which consists of developing websites, apps, and other digital products for brands. We must also focus on creating a good funnel for the services. Try to take them from one service to another.

We are transforming it from a branding-focused site into a structured, product-like platform.

Core Concept:
Centauri becomes a “Growth Platform” where:

* Creators grow their audience and access opportunities
* Brands discover and collaborate with creators
* Solutions service for brands to develop their own websites, apps, and other digital products
* Funnel for the services
* Clear user journeys
* Strong CTA hierarchy
* Conversion-focused structure
* Clear hierarchy and spacing
* No unnecessary elements
* Every screen must guide user action


Primary Fixes:

1. Productize the experience:

   * Dashboard UI
   * Creator cards
   * Analytics (mock data)
   * Opportunities section

2. Add micro-interactions:

   * Hover states
   * Transitions
   * Animated numbers (simple, not heavy libraries)

3. Add proof/credibility:

   * Metrics (mock)
   * Case-style sections
   * Growth indicators

Tech Stack:

* Next.js (App Router)
* TypeScript
* Tailwind CSS
* shadcn/ui
* Framer Motion
* Zustand (light state management)
* React Hook Form + Zod (or native FormData)

No backend:

* Use mock data (static JSON or local constants)
* No API integration required

Pages to build:

Shared Components

* Navbar with persistent CTA
* Buttons with clear hierarchy
* Cards (creator, metrics)
* Sections (reusable layout blocks)

Success Criteria:

* Clear user flows
* Strong CTA hierarchy
* Feels like a real product, not just a landing page
* Demonstrates improved engagement and conversion logic

Important:
Do not over-engineer.
Focus on clarity, structure, and UX over complexity.

We are creating for engagement and conversion.

Important:

Think as a designer and a developer, we need to design the funnel and the components, considering in every moment that we want to transmit what centauri is and what they do. Think about the user journey and the user experience.

Always follow best practices and design patterns.
